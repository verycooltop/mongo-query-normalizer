# npm run test 失败用例分类盘点（字段基数 / 保守语义）

本文件用于归类：哪些失败来自**旧版「单值字段」契约**（在未知字段基数下对 MongoDB 多键/数组语义不成立），以及应如何对待测试。

## 原则（当前主契约）

- **未知字段基数** → 一律按**保守行为**建模：默认**不做 schema 假设**。
- 仅执行**可证明安全**的重写；不依赖「同一标量元素同时满足多个约束」的推理。
- **旧单值语义契约已废弃**：相关测试应改写为保守语义下的正确预期，或删除过时断言，而不是再维护可切换的「激进模式」或并行的第二套语义套件。

## 历史表格（背景）

此前部分用例期望在单字段上折叠矛盾、`$in` 与 bounds 求交、多 `$in` 求交等。对可能为数组的字段，这些期望在 MongoDB 语义下**不成立**，已从实现与测试中剥离。

若 triage 时遇到仍引用上述旧契约的失败用例，请按上文原则改为保守预期（不判死、不强求交、不激进收紧），并补充「幂等 / 语义稳定 / 不做不安全收紧」类正向断言。
